# Fastq Merger & Demultiplexer (Xpress)

**Xpress Fastq Merger** 是一个专为高通量测序数据设计的处理工具。它主要用于“反向拆分合并”（Reverse Demultiplexing/Merging）场景：读取一份样本清单（Sample Sheet），从海量测序文件中根据 ID 抓取对应的 R1/R2 文件，对 Read 2 序列进行特定的 Barcode 标记处理，最后将所有样本合并为一对主文件。

该工具基于 Rust 语言开发，核心设计理念是**安全**、**极速**和**自动化**。

---

## 🌟 核心特性

### 1. 极致性能
*   **并行加速 (Multithreading)**：基于 `rayon` 框架，能够自动检测服务器 CPU 核心数，并行处理多个样本。对于海量样本合并任务，速度随核数线性提升。
*   **高效 I/O**：内部采用 64KB 缓冲区与流式处理（Streaming），内存占用极低（仅几 MB），即使处理 TB 级数据也不会爆内存。
*   **Map-Reduce 架构**：每个线程独立生成临时分块，最后极速拼接（Zero-compression overhead），避免了写入锁竞争。

### 2. 智能与自动化
*   **自动配对**：你只需提供 ID（如 `10`），程序会自动在文件夹中搜索符合 `_10_` 或 `-10-` 模式的文件，并智能区分 R1 和 R2。
*   **严格过滤**：内置白名单机制，自动忽略 `.txt`, `.json`, `.html` 等干扰文件，只处理 `.fastq` / `.fq` 及其 `.gz` 压缩包。
*   **环境自适应**：输出目录不存在？自动创建。输出文件已存在？报错保护。

### 3. 数据完整性守门员
程序不仅仅是合并文件，更是数据的“质检员”：
*   **文件截断检测**：如果 FASTQ 文件末尾缺失（下载中断），程序会立即捕获错误。
*   **配对同步检测**：严格比对 R1 和 R2 的行数，确保 Reads 一一对应。
*   **ID 匹配校验**：
    *   **严格模式**：检查 R1 和 R2 的 Read ID 是否一致。
    *   **兼容模式**：自动识别（R1 结尾 `/1`，R2 结尾 `/2`），不会误报 ID Mismatch。
*   **事务性（Transactional）**：**全有或全无**。只要有一个样本处理失败，程序会取消合并并清理所有临时文件，绝不生成“缺斤少两”的坏数据。

## 📖 详细使用指南

### 命令行参数

```bash
fastq_merger [OPTIONS] --input-dir <INPUT_DIR> --samplesheet <SAMPLESHEET>
```

| 参数 | 简写 | 必填 | 默认值 | 详细说明 |
| :--- | :--- | :--- | :--- | :--- |
| `--input-dir` | `-i` | ✅ | - | **输入目录**。程序会扫描该目录（不递归子目录）下的所有文件。 |
| `--samplesheet` | `-s` | ✅ | - | **样本表**。CSV 格式，定义了要提取哪些样本。 |
| `--output-dir` | `-o` | ❌ | `output` | **输出目录**。程序会自动创建此目录。最终文件和临时文件都存放在这里。 |
| `--compress` | `-c` | ❌ | (关) | **开启压缩**。加上此标记后，输出文件为 `.fastq.gz`，否则为纯文本 `.fastq`。 |
| `--threads` | `-t` | ❌ | 0 | **线程数**。默认为 `0`（自动跑满 CPU）。如果服务器有负载限制，可手动设为 `4` 或 `8`。 |

### 📂 输入文件准备

#### 1. Sample Sheet (样本表)
这是程序的“指挥棒”。支持 CSV (逗号) 格式。

*   **结构要求**：至少两列。
    *   **Column 1 (ID)**: 样本编号。用于匹配文件名。
    *   **Column 2 (Seq)**: 条形码序列。将被拼接到 R2 Read 的序列末尾，质量值填充为 `I`。
*   **去重**：程序运行前会自动检查 ID 和 Seq 是否有重复，如有重复直接报错退出。
*   **表头**：程序会自动忽略包含 "id" 或 "seq" (不区分大小写) 的第一行。

**示例 `samplesheet.csv`:**
```csv
Barcode_ID,Sequence
10,ATGCATGC      <-- 对应文件如 Library_10_R1.fq.gz/Library_10_R2.fq.gz
11,CGTACGTA      <-- 对应文件如 Library_11_R1.fq.gz/Library_11_R2.fq.gz
```

#### 2. 原始 FASTQ 文件
文件名必须遵循一定的命名规范，以便程序能找到它。

*   **后缀要求**：必须是 `.fastq`, `.fq`, `.fastq.gz`, `.fq.gz` 之一。
*   **ID 匹配规则**：文件名必须包含 `_ID_` 或 `-ID-`。
    *   例如 ID 为 `10`：
    *   ✅ `Sample_10_L01_R1.fq.gz` (匹配 `_10_`)
    *   ✅ `Lib-10-Rep1_R1.fq` (匹配 `-10-`)
    *   ❌ `Sample10.fq` (缺少分隔符，容易误匹配)
*   **R1/R2 区分规则**：
    *   **R1**: 文件名包含 `_1.` 或 `_R1.` 或 `.R1.`
    *   **R2**: 文件名包含 `_2.` 或 `_R2.` 或 `.R2.`

## 🚀 运行示例

### 场景 A：常规处理 (推荐)
读取 `raw_data` 目录，根据 `samples.csv` 合并，输出压缩文件到 `merged_results` 目录，使用所有 CPU 核心。

```bash
./fastq_merger \
  --input-dir ./raw_data \
  --samplesheet ./samples.csv \
  --output-dir ./merged_results \
  --compress
```

**输出结果**：
*   `merged_results/demultiplexed.R1.fastq.gz`
*   `merged_results/demultiplexed.R2.fastq.gz`

### 场景 B：调试模式 (少量线程)
如果您在共享服务器上，不想占用太多资源，可以限制线程数，且不压缩输出（方便查看内容）。

```bash
./fastq_merger \
  -i ./raw_data \
  -s ./samples.csv \
  -o ./debug_out \
  --threads 4
```

## ❓ 常见报错与解决

| 错误信息 (Error) | 原因分析 | 解决方案 |
| :--- | :--- | :--- |
| `Duplicate Barcode ID found` | Sample Sheet 里有重复的 ID。 | 检查 CSV 文件，删除重复行。 |
| `Could not find pair for barcode ID '10'` | 在输入目录找不到 ID 为 10 的文件，或者文件名格式不对（如缺少 `_` 分隔符）。 | 确认目录下是否有 `_10_` 命名的文件，且后缀正确。 |
| `ID Mismatch for 10. R1: 'seqA', R2: 'seqB'` | 同一行的 R1 和 R2 Read ID 不一致，说明文件内容不同步。 | 检查源文件是否损坏或被错误配对。 |
| `Mismatch/Truncated ID 10` | 读取过程中，一个文件结束了，另一个还有数据。 | 源文件可能下载中断或被截断。请重新下载原始数据。 |
| `Output files already exist` | 输出目录下已经有 `demultiplexed.*` 文件。 | 为防止覆盖，程序拒绝运行。请删除旧文件或更换输出目录。 |

## ⚙️ 技术细节 (处理逻辑)

1.  **预扫描**：程序启动时，先将输入目录下的所有合法 FASTQ 文件路径加载到内存索引中（避免每次遍历磁盘）。
2.  **验证表单**：读取 Sample Sheet，检查是否有重复 ID/Seq。
3.  **并行任务分发**：
    *   Master 线程根据 ID 分配任务。
    *   Worker 线程领取 ID，从索引中找到 R1/R2 路径。
    *   Worker 线程边读边写（Streaming），生成临时的分块文件（如 `tmp_0_10.R1.fastq.gz`）。
    *   Worker 线程处理 R2 时，会实时修改序列和质量值。
4.  **汇总与合并**：
    *   所有任务完成后，统计成功率。
    *   如果有 **任何失败** -> 删除所有临时文件 -> 报错退出。
    *   如果 **全部成功** -> 按 Sample Sheet 顺序将临时文件二进制拼接到主文件 -> 删除临时文件 -> 完成。
